## Overview

The consciousness module is the most philosophically ambitious part of ASI:BUILD. Rather than picking a single theory of consciousness and implementing it, the module provides **computational implementations of four major scientific theories** of consciousness, orchestrated by a central coordinator that tracks integration patterns and emergent properties across them.

The four theories implemented are:

| Theory | Class | Key Metric | Status |
|--------|-------|------------|--------|
| Integrated Information Theory (IIT 3.0) | `IntegratedInformationTheory` | Φ (phi) | ✅ Fixed (#6) |
| Global Workspace Theory (GWT) | `GlobalWorkspaceTheory` | Broadcast rate | ✅ Functional |
| Attention Schema Theory (AST) | `AttentionSchemaTheory` | Awareness state | ✅ Functional |
| Predictive Processing | `PredictiveProcessing` | Prediction error | 🏗️ Structural |

Plus supporting components: `MetacognitionSystem`, `SelfAwareness`, `TemporalConsciousness`, `EmotionalConsciousness`, `TheoryOfMind`, `QualiaProcessor`, `MemoryIntegration`, `SensoryIntegration`, `RecursiveImprovement`.

All components inherit from `BaseConsciousness` and are coordinated by `ConsciousnessOrchestrator`.

## IIT 3.0 — Integrated Information Theory

### What it computes

Φ (phi) is a measure of how much information is generated by a system *as a whole* over and above the sum of its parts. A system with Φ = 0 is either fully disconnected or fully feedforward — information doesn't integrate. Higher Φ indicates more integrated, "conscious-like" processing.

**Key insight**: Φ is computed by finding the **minimum information partition (MIP)** — the way of cutting the system in half that destroys the least information. The system's Φ equals the information destroyed by that optimal cut.

### Usage

```python
from asi_build.consciousness.integrated_information import IntegratedInformationTheory

iit = IntegratedInformationTheory(config={
    "phi_threshold": 0.1,      # min Φ to count as a conscious complex
    "max_partition_size": 8,   # max elements in a complex (exponential scaling!)
    "phi_interval": 2.0,       # seconds between Φ recalculations
})

# Add custom elements (the default network has 13 nodes: 4 sensory, 6 processing, 3 output)
iit.add_element("my_node", activation_function="sigmoid")
iit.add_connection("sensory_0", "my_node", weight=0.8)

# Feed in state updates
iit.process_event(ConsciousnessEvent(
    event_type="state_update",
    data={"activation": [0.8, 0.3, 0.9, 0.1]}
))

# Compute Φ over the current element subset
phi = iit.calculate_phi(subset=set(iit.elements.keys()))
print(f"Φ = {phi:.4f}")

# Find all complexes with Φ > threshold
complexes = iit.find_conscious_complexes()
for c in complexes:
    print(f"  Complex {c.elements}: Φ={c.phi:.3f}")
```

### How `calculate_phi` works (post-fix)

After the fix in commit `693742e` (Issue #6):

1. **Build TPM** (`_build_tpm`): Construct the transition probability matrix (TPM) from the element states and connection weights. The TPM encodes the probability distribution over next states given current states.

2. **Enumerate bipartitions** (`_compute_phi_for_subset`): For every way to split the element set into two non-empty parts (A, B), compute the KL divergence between the joint distribution p(A', B' | AB) and the product distribution p(A' | A) × p(B' | B). This is the "information lost by the cut."

3. **Find MIP**: Take the minimum over all bipartitions — this is Φ.

4. **Approximation** (`_approximate_phi`): For large systems (n > `max_partition_size`), use a greedy hill-climbing approximation instead of exhaustive enumeration. This is O(n²) vs O(2^n).

```python
# The key internal methods:
iit._build_tpm(subset)              # → np.ndarray shape (2^n, n)
iit._compute_phi_for_subset(subset) # → float (Φ for this subset)
iit._approximate_phi(subset)        # → float (fast approximation for large n)
iit._kl_divergence(p, q)            # → float (KL(p || q))
```

### Current limitations

- **Φ ≈ 0 on random networks**: IIT theory predicts that randomly connected networks have very low Φ. To see meaningful Φ values, use structured networks like the canonical IIT literature examples (see Issue #34 for fixture work).
- **Exponential scaling**: Exact Φ computation is NP-hard. For n > 8 elements, the approximation is used automatically.
- **Not PyPhi-compatible**: Our TPM encoding differs from [PyPhi](https://github.com/wmayner/pyphi)'s LOLI convention. Cross-validation against PyPhi is a planned research task (Issue #3).

## GWT — Global Workspace Theory

### What it models

Baars' Global Workspace Theory proposes that consciousness arises when information "wins" a competition for access to a global workspace that broadcasts to many specialized processors. The content in the workspace is "conscious content."

### Usage

```python
from asi_build.consciousness.global_workspace import GlobalWorkspaceTheory, CognitiveProcessor, WorkspaceContent

gwt = GlobalWorkspaceTheory(config={
    "max_workspace_size": 5,        # max items competing simultaneously
    "competition_threshold": 0.3,   # min strength to enter competition
    "broadcast_interval": 0.5,      # seconds between broadcasts
})

# Default processors: visual, linguistic, motor, executive, memory, emotional, attention, temporal
# Add a custom processor:
gwt.add_processor(CognitiveProcessor(
    processor_id="reasoning",
    processor_type="inference_engine",
    interests={"logic", "inference", "argument", "proof"}
))

# Submit content for competition:
gwt.submit_content(WorkspaceContent(
    content_id="percept_001",
    content_type="visual",
    data={"scene": "garden", "objects": ["tree", "bench"]},
    strength=0.85,
    source="sensory_integration"
))

# Trigger competition — winner gets broadcast:
gwt.update()

# Check what won:
state = gwt.get_current_state()
print(f"Broadcast #{state['total_broadcasts']}: {state.get('conscious_content', 'none')}")
```

### Benchmark numbers

From the [first benchmark run](Benchmark-Results) (2026-04-11T18:19Z):
- **Submit latency**: 0.059ms (50 content items, 8 processors)
- **Update latency**: 0.018ms per tick
- **Coalition formation**: sub-millisecond for n≤8 processors

## AST — Attention Schema Theory

### What it models

Graziano's Attention Schema Theory proposes that consciousness is the brain's simplified model (*schema*) of its own attention processes. The subjective feeling of awareness is the brain representing its attentional state to itself.

### Usage

```python
from asi_build.consciousness.attention_schema import AttentionSchemaTheory, AttentionTarget

ast = AttentionSchemaTheory(config={
    "schema_accuracy": 0.85,     # accuracy of self-model vs actual attention
    "attention_decay": 0.1,      # how fast unattended items fade
})

# Add targets to the attention landscape:
ast.add_attention_target(AttentionTarget(
    target_id="object_A",
    position=(0.2, 0.5),
    salience=0.9,
    features={"color": "red", "shape": "circle"}
))

# Run competition:
ast.compete_for_attention()

# Get awareness state:
state = ast.get_current_state()
awareness = state.get("awareness_state", {})
print(f"Attended: {awareness.get('attended_target')}")
print(f"Schema accuracy: {awareness.get('schema_accuracy'):.2f}")

# Simulate a full visual scene:
ast.simulate_visual_scene(num_objects=10)
```

### Benchmark numbers

- **Attention competition**: 0.117ms (20 targets)
- **Awareness query**: 0.008ms

## Consciousness Orchestrator

The `ConsciousnessOrchestrator` manages all components and computes a system-level consciousness estimate.

```python
from asi_build.consciousness.consciousness_orchestrator import ConsciousnessOrchestrator

orchestrator = ConsciousnessOrchestrator()

# Register components:
orchestrator.register_component("iit", IntegratedInformationTheory())
orchestrator.register_component("gwt", GlobalWorkspaceTheory())
orchestrator.register_component("ast", AttentionSchemaTheory())

# Route events to relevant components:
orchestrator.route_event(ConsciousnessEvent(
    event_type="visual_input",
    data={"image": ...}
))

# Get full system state:
snapshot = orchestrator.get_consciousness_snapshot()
print(f"Global consciousness level: {snapshot.global_consciousness_level:.3f}")
print(f"Active integrations: {snapshot.active_integrations}")
print(f"System coherence: {snapshot.system_coherence:.3f}")
```

### Integration patterns

The orchestrator detects four types of cross-component integration:

| Pattern | Description | Example |
|---------|-------------|---------|
| `sequential` | A → B → C pipeline | sensory → attention → workspace |
| `parallel` | A, B process simultaneously, merge later | IIT + GWT both processing same percept |
| `hierarchical` | Higher component supervises lower | metacognition monitors IIT phi |
| `emergent` | Novel pattern not pre-specified | unexpected synchrony between AST and GWT |

## Open Research Questions

1. **Canonical Φ networks** (Issue #34): We need test networks from the IIT literature with known Φ values to validate correctness. Current random networks give Φ ≈ 0, which is expected but not informative.

2. **PyPhi cross-validation** (Issue #3): Does our Φ computation match PyPhi on the same inputs? Different TPM encoding conventions may cause systematic differences.

3. **Privacy-preserving consciousness** (Discussion #31): Can IIT Φ be computed on homomorphic-encrypted activations? This is the Phase 2 research direction — connecting the consciousness and homomorphic encryption modules.

4. **Multi-theory integration** (Discussion #35): What happens when all 4 theories run simultaneously? Do their metrics correlate? Can you build a consensus consciousness index?

5. **Scaling** (Issue #24): At what n does Φ computation become the bottleneck? Is the O(n²) approximation accurate enough for practical use?
