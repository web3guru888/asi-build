# ASI-Code GraphQL Schema via Hasura

## Overview
Complete GraphQL API exposing ALL PostgreSQL data with real-time subscriptions.

## Access Points
- **GraphQL Endpoint:** http://localhost:8080/v1/graphql
- **Hasura Console:** http://localhost:8080/console
- **WebSocket (Subscriptions):** ws://localhost:8080/v1/graphql
- **Admin Secret:** asi_hasura_admin_secret_2024

## Table Relationships Graph

```
sessions (root)
├── conversations (1:N)
│   ├── messages (1:N)
│   └── api_calls (1:N)
├── orchestrations (1:N)
│   ├── tasks (1:N)
│   │   ├── subtasks (self-referential)
│   │   └── generated_files (1:N)
│   └── projects (1:N)
│       └── files (1:N)
├── projects (1:N)
├── websocket_connections (1:N)
│   └── messages (1:N)
├── system_logs (1:N)
└── performance_metrics (1:N)

agents (standalone)
api_usage (standalone)
retry_attempts (standalone)
```

## Key GraphQL Queries

### 1. Get Session with All Related Data
```graphql
query GetSessionComplete($sessionId: String!) {
  sessions(where: {session_id: {_eq: $sessionId}}) {
    id
    session_id
    ip_address
    created_at
    is_active
    
    # All conversations
    conversations {
      conversation_id
      model_used
      total_messages
      total_tokens_used
      
      # All messages in conversation
      messages(order_by: {message_index: asc}) {
        role
        content
        tokens_used
        created_at
      }
      
      # API calls for this conversation
      api_calls {
        endpoint
        response_status
        response_time_ms
        error_message
      }
    }
    
    # All orchestrations
    orchestrations {
      orchestration_id
      task_description
      status
      total_tasks
      completed_tasks
      
      # All tasks in orchestration
      tasks(order_by: {id: asc}) {
        task_id
        name
        assigned_agent
        status
        can_parallel
        dependencies
        result
        
        # Files generated by task
        generated_files {
          file_name
          file_path
          content
          file_size_bytes
        }
      }
    }
    
    # All projects
    projects {
      project_name
      project_type
      framework
      total_files
      
      # All files in project
      files {
        file_name
        file_path
        content
        language
      }
    }
  }
}
```

## Mutations

### 1. Update Task Status
```graphql
mutation UpdateTaskStatus($taskId: String!, $status: String!, $result: String) {
  update_tasks(
    where: {task_id: {_eq: $taskId}}
    _set: {
      status: $status
      result: $result
      completed_at: "now()"
    }
  ) {
    affected_rows
    returning {
      task_id
      status
      result
    }
  }
}
```

### 2. Create System Log
```graphql
mutation LogEvent($level: String!, $source: String!, $message: String!, $context: jsonb) {

    message: $message
    context: $context
  }) {
    id
    created_at
  }
}
```

## Apollo Client Integration

### Installation
```bash
npm install @apollo/client graphql subscriptions-transport-ws
```

### Client Setup
```typescript
import { ApolloClient, InMemoryCache, split, HttpLink } from '@apollo/client';
import { WebSocketLink } from '@apollo/client/link/ws';
import { getMainDefinition } from '@apollo/client/utilities';

// HTTP connection to the API
const httpLink = new HttpLink({
  uri: 'http://localhost:8080/v1/graphql',
  headers: {
    'x-hasura-admin-secret': 'asi_hasura_admin_secret_2024'
  }
});

// WebSocket connection for subscriptions
const wsLink = new WebSocketLink({
  uri: 'ws://localhost:8080/v1/graphql',
  options: {
    reconnect: true,
    connectionParams: {
      headers: {
        'x-hasura-admin-secret': 'asi_hasura_admin_secret_2024'
      }
    }
  }
});

// Split links based on operation type
const splitLink = split(
  ({ query }) => {
    const definition = getMainDefinition(query);
    return (
      definition.kind === 'OperationDefinition' &&
      definition.operation === 'subscription'
    );
  },
  wsLink,
  httpLink
);

// Apollo Client
export const apolloClient = new ApolloClient({
  link: splitLink,
  cache: new InMemoryCache()
});
```

### React Hook Usage
```tsx
import { useSubscription, gql } from '@apollo/client';

const TASK_SUBSCRIPTION = gql`
  subscription WatchTasks($orchestrationId: String!) {
    tasks(where: {
      orchestration: {orchestration_id: {_eq: $orchestrationId}}
    }) {
      task_id
      name
      status
      assigned_agent
    }
  }
`;

function TaskMonitor({ orchestrationId }) {
  const { data, loading, error } = useSubscription(TASK_SUBSCRIPTION, {
    variables: { orchestrationId }
  });

  if (loading) return <div>Loading...</div>;
  if (error) return <div>Error: {error.message}</div>;

  return (
    <div>
      {data.tasks.map(task => (
        <div key={task.task_id}>
          {task.name} - {task.status} ({task.assigned_agent})
        </div>
      ))}
    </div>
  );
}
```

## Performance Optimization

### 1. Indexes (Already Created)
- All foreign keys indexed
- Timestamp columns indexed
- Status fields indexed

### 2. Query Optimization
- Use `limit` for large datasets
- Leverage aggregations for counts
- Use subscriptions sparingly

### 3. Caching
- Enable query caching in Hasura
- Use Apollo Client cache
- Implement Redis for heavy queries
